Week Five Notes

Stack traces identify moving backwards where the issue originated and lists the steps in between. strict mode is used to alert for any basic bad practice errors in js. The best way to use strict mode is to put all of your code into a self-invoking function.

Linting tools are used to highlight sloppy code. This helps create order and contiguity when multiple people are working in the same code. Additionally, it creates a consistent style throughout the code. Plugins can be created to force people to code using specific styles and syntaxes when multiple options will do.

Using the alert() method is good for testing. You can also use console.log() for similar purposes, but alert() will stop the program whereas console.log() will not. Console.table() will build a table with the data in the console. Console.trace() will log a stack trace.

The debugger keyword can be implemented in code to stop the program (it works like a breakpoint) and it is interactive, meaning you can see what variables currently are at that point in the code. It is important to remove debugger code before deployment or else the code will freeze for the users.

You can create a custom error() with the error function. Additionally, there are seven more built in errors in JS. These are all thrown automatically. To throw custom exceptions you would use the throw keyword.

When errors are thrown there are ways to handle them so that they don't impede the user. To do this you would need to use try, catch, and finally.
